UI Renderer (Firmware Stub)

Overview
- Files: src/ui_render.h, src/ui_render.c
- Purpose: Render exported scenes (ui_design.h/ui_design.c) using backend-agnostic drawing callbacks.

How it works
- Provide UiDrawOps with callbacks for fill_rect, draw_hline, draw_vline, draw_rect, draw_text (any subset; the renderer falls back when possible).
- Call ui_render_scene(&UI_SCENE_DEMO, &ops) (or any exported scene) to draw widgets.
- Integrate the callbacks with your framebuffer or display driver.

New: Software framebuffer adapter

- Files: src/ui_render_swbuf.h, src/ui_render_swbuf.c
- Use packed 1bpp buffer with stride = (W+7)/8. Create draw ops with ui_swbuf_make_ops().
- Set pixel format in src/display_config.h via DISPLAY_COLOR_BITS (1 or 4).
- Flush helpers:
    - Full frame: ui_swbuf_flush_auto_ssd1363()
    - Dirty rect: ui_swbuf_flush_dirty_auto_ssd1363()
    - Explicit 1bpp: ui_swbuf_flush_ssd1363()/ui_swbuf_flush_dirty_ssd1363()
    - Explicit 4bpp: ui_swbuf_flush_gray4_ssd1363()/ui_swbuf_flush_dirty_gray4_ssd1363()

Quick example (pseudo-code)

#include "display/ssd1363.h"
#include "ui_design.h"      // generated by ui_export_c.py
#include "ui_render.h"

#define W 128
#define H 64
static uint8_t fb[W * H]; // 1 byte per pixel (example), pack as needed

static void fb_fill_rect(void *ctx, int x, int y, int w, int h, uint8_t c) {
    for (int yy = 0; yy < h; ++yy) {
        for (int xx = 0; xx < w; ++xx) {
            int px = x + xx, py = y + yy;
            if (px < 0 || py < 0 || px >= W || py >= H) continue;
            fb[py * W + px] = c ? 0xFF : 0x00; // simple 8bpp example
        }
    }
}

static void fb_hline(void *ctx, int x, int y, int w, uint8_t c) {
    fb_fill_rect(ctx, x, y, w, 1, c);
}

static void fb_vline(void *ctx, int x, int y, int h, uint8_t c) {
    fb_fill_rect(ctx, x, y, 1, h, c);
}

static void fb_rect(void *ctx, int x, int y, int w, int h, uint8_t c) {
    fb_hline(ctx, x, y, w, c);
    fb_hline(ctx, x, y + h - 1, w, c);
    fb_vline(ctx, x, y, h, c);
    fb_vline(ctx, x + w - 1, y, h, c);
}

static void fb_text(void *ctx, int x, int y, const char *text, uint8_t c) {
    // optional: integrate a tiny bitmap font. For now, draw a simple marker line.
    fb_hline(ctx, x, y, (int)strlen(text), c);
}

void app_render_demo(void) {
    UiDrawOps ops = {0};
    ui_swbuf_make_ops(&sw, &ops); // or set callbacks manually as above

    // 1) init panel
    ssd1363_bus_init();
    ssd1363_init_panel();

    // 2) clear framebuffer
    memset(fb, 0x00, sizeof(fb));

    // 3) render scene exported by ui_export_c.py
    ui_render_scene(&UI_SCENE_DEMO, &ops);

    // 4) flush fb to panel (auto-select 1bpp/4bpp based on DISPLAY_COLOR_BITS)
    ui_swbuf_flush_auto_ssd1363(&sw);
}

Notes
- ui_design.h/ui_design.c are generated by running: D:\ESP32OS\.venv\Scripts\python.exe d:\ESP32OS\ui_export_c.py
- The SSD1363 init in src/display/ssd1363.c is a placeholder. You’ll need to send a correct init sequence and implement the RAM write (addressing mode, column/page ranges) for your module.
- The renderer uses monochrome color (0/1). Extend UiDrawOps and UiWidget (and exporter) if you add grayscale/color.
